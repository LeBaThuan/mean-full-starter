Deploying to Now

Zeit Now is a company that build software and cloud products for developers and designers. Most notably, they're the company behind Next.js, which is the server side rendering platform for React. Their goal is to make cloud computing easy, universal, and accessible. Now's platform is likely one of the easiest for app deployment. All it takes to deploy your app to the cloud is by using the command now, no Git repository needed! They offer multiple pricing tiers based on usage, including a free tier called OSS. You can check out their pricing plans here. In a nutshell, all deployments are final, and if you ever have to update a feature in your app, you simply deploy the app by using the now command and a new app and URL will appear instantly. This guide explains the basic principles of deploying your app to Now.

Deploying to Now

  1. Install Now Desktop at Zeit Now's download page.
  2. Ensure all links in Angular services are changed from absolute links to relative links (example shown below)
    3.                 // book.service.ts
                getAllBooks(): Observable<Book[]> {{ '{' }}
                  return this.http.get<Book[]>('http://localhost:3000/api/library/book');  // before
                {{ '}' }}
              
                    // book.service.ts
                getAllBooks(): Observable<Book[]> {{ '{' }}
                  return this.http.get<Book[]>('api/library/book');  // after
                {{ '}' }}
  3. Ensure the correct scripts are loaded in package.json. After installing npm packages, Now executes the build command, followed by the start command. If aliases are needed, you could also use now-build or now-start.
    4.                     // package.json
                    "scripts": {{ '{' }}
                      "build": "ng build --prod && ng build --prod --app universal --output-hashing=none",
                      "start": "node server.js"
                     {{ '}' }},
  4. Deploy your app by typing in now in the command line.
    5. now
    The app will build and logs will be shown in your command line. If you are using the free tier, please keep in mind that your source code will also be public and easily accessible by adding _src to the end of the root route URL (eg. https://www.example.com/_src). You have now successfully deployed the app!
